Although I have been using the web since the late 90s, web development is something I personally do not have much experience with... unless one counts setting up a page on Geocities in 1997. S06

A cursory search of which programming language is the most popular or most used shows that there does not appear to be any consensus on the popularity of programming languages. TIOBE says it’s Java followed by C and then Python, while Octoverse say it’s JavaScript followed by Python, then Java. When dealing exclusively with of server-side programming languages for websites, W3Techs states PHP is overwhelmingly the most used, but Google Trends currently puts in below Python, Java, JavaScript, and C.

Looking at Google Trends over the last five years, the ‘interest over time’ index worldwide for C, PHP and JavaScript all remain relatively consistent. The two standouts here are Java, which reigned supreme in 2015 by a sizable margin, but had its lead gradually eroded by Python until early 2019, when the lead changed hands. It does seem that Python is going to be the programming language of the future as it is the only one that shows consistent growth over the last five years.

If I were to recommend a language/framework/technology to a potential employer for my next large project, my recommended language would have to be either Java or Python. Python may be the coming thing and experience with it is highly sought after in the job market, but Java is still widely used, consistently updated and one of the easier languages to learn.

S07

Here we have three three topics of .NET Core, C# and ASP.NET. The .NET Core is a extensive platform with a ton of potential uses since it runs in so many different environments. It does kind make me wonder why we haven't been learning .NET along side Java for the past two years.

C# is is a general-purpose, multi-paradigm programming language. It encompasses strong typing, lexically scoped, imperative, declarative, functional, generic, class-based object-oriented, and component-oriented programming disciplines. C# is one of the programming languages designed for the Common Language Infrastructure (CLI). Again, it is a little frustrating that we're only touching on such a fundamental programming language after spending more than two years with Java.

ASP.NET is a server-side web-application framework designed for web development made to allow programmers to build dynamic web sites, applications and services. ASP.NET is built on the Common Language Runtime (CLR), allowing programmers to write ASP.NET code using any supported .NET language such as C#. This, much like the other server-side technologies we have learned, is extremely challenging to learn for for the layman.

The approach I took for this assignemnt was similar to the one I took with A04 and A05.

S08

In choosing three wbesites to analyse, tried to choose pages with three vastly different purposes. To that end, I chose:

  • A news site
  • An online store
  • A tech-basded tutorial site

The tools I used to analyse the site performance were Lighthouse, PageSpeed Insights and my browser's devtool tab. I conduceted this research on my own.

The Age news site

The Age is a news site from Melbourne, Australia. The site scored very low scores for performance on both Lighthouse and PageSpeed Insights. By both measures, the page takes several seconds to fully load and has a speed indes of more than 10 seconds.



This is further highlighted in by the network information in the devtools showing that this page took nealry 10 seconds to load completely. The reason for this appears to be the sheer volume and size of the images being processed: the ones highlighted here ranged approximately between 200KB to 800KB. The page iteself is also extremlely long and refereshes itself periodically.


Dressmann XL online store

Dressmann XL is an online men's clothing store. The site scored low scores for performance on both Lighthouse and PageSpeed Insights, although not as low as The Age. By both measures, the page takes several seconds to fully load but, curiously, The Age took less time time to load at first, although overall Dressmann XL performed better.


The page's average performance can be explained the network information in the devtools, showing that this page took nealry 3 seconds to load completely. The reason for this appears to be similar to The Age, that being the volume and size of the images being processed.


w3schools web development tutorials

w3schools provides tutorials, resources and many services to anyone wanting to learn web development. The site scored highly scores for performance on both Lighthouse and PageSpeed Insights, especially when compared to the other two sites previously mentioned. By both measures, the page has a speed index of less than half of both Dressmann XL and The Age (2.6 seconds compared to The Age's 11 seconds).


The page's average efficiency can be explained the network information in the devtools, showing that this page's size and volume of pictures are relatively small, instead relying on text and decived such as sidebars containing dozen of links to other parts of the site rather than using one large page like The Age does.

I found this section paticualrly interesting as it is an branch of study I plan to follow when (if) I graduate. Namely, the field of network forensics. Being a veteran user of the internet, website security is something I often considering when using the web, but learning about it from the development is a real eye-opener. Up until now, the only security measures we implemented as part of application development was a three-chance login system and two-factor authentication. We did study a unit in Computer Security, but I really have liked to spend more time in this area of study, the being website secuirty from a developer's perspective in particular.

Another aspect I found interesting was the OWASP Top Ten list, especially when comparing the list from 2013 to 2017 and today.

Some security issues or concerns from your my own experience:

  • Input sanitsing - This is something I don't usual consider when designing an applicaiton or lately a website, espeically considering 'injection flaws' has remained on the top of OWASP's Top Ten list since at least 2013.
  • Using components with known vulnerabilities - At this stage of our programming experience, we are proabably more concerned with finding components and APIs that acutally work the way we want them to rather than being concerned with how secure they are.
  • Insufficient logging and monitoring - As with the previous issues, this is something I have not considered when building applicaitons and website during the course. Indeed, unless an attack came that was obvious, such as changing an interface, recognising and resposnding to an attack would be something I would be largely unprepared for. I would have really liked to exlpore this more in this unit.

TIL:

  • Abiut OWASP and their historic Top Ten securtiy vulnerabilities
  • Fundamentals of the Top Ten security vulnerabilities and what impact they could have if exploited
  • Commonly overlooked security fundamentals and what hackers look for to be able to exploit these flaws
  • There is a lot more to consider when building a website apart from aesthetics
